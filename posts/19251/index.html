入口：

@Import(TransactionManagementConfigurationSelector.class)
public @interface EnableTransactionManagement {

public class TransactionManagementConfigurationSelector extends AdviceModeImportSelector {
@Override
protected String[] selectImports(AdviceMode adviceMode) {
switch (adviceMode) {
// 绝大部分情况，步回使用AspectJ 的静态代理
// 导入下面两个类
case PROXY:
return new String[] {AutoProxyRegistrar.class.getName(),
ProxyTransactionManagementConfiguration.class.getName()};
case ASPECTJ:
return new String[] {determineTransactionAspectClass()};
default:
return null;
}
}

解析切面：
createBean 第一次创建bean的时候，解析切面
beanDefinitionMap object.class
@AspectJ
@Before
@After

advisor -> advice + pointcut

创建动态代理：
createBean
doCreateBean
initionalizion
afterprocessor
advisors -> bean match 匹配，aspectJ api，加入集合

aopProxyFactory:
接口 -> jdk
非接口 -> cglib
设置proxyTargetClass = true，强制使用cglib

jdk 和cglib 动态代理区别？ 效率上的区别，极其微小
jdk 代理的是接口， .class的字节码文件(实现原理：实现了具有代理的接口)
根据接口生成一个proxy.class(动态代理文件，jdk底层实现)，调用具体方法 -> 使用反射
增强代码，反射 -> 调用被代理的方法

cglib，指定的是实现类，不需要指定接口(底层采用asm字节码框架去创建的.class文件，创建多个)
创建过程，jdk动态代理更快，因为文件更少
cglib动态代理类，使用的是继承(看字节码文件)
根据具体的类，生成多个cglib.class，使用继承方式
本类调用其他方法，会重复增强，
动态代理类，继承了被增强的类，直接调用
